01 / 소개
WHAT

바이브 코딩이란
무엇인가?

2025년 2월, OpenAI의 공동창업자 안드레이 카르파티(Andrej Karpathy)가 처음 제시한 개념입니다. 그는 "나는 완전히 바이브에 몸을 맡기고, 코드를 거의 보지 않으며, AI가 뭔가 잘못됐다고 말할 때조차 그냥 시도해본다"라고 말했습니다.

전통적인 개발은 언어를 배우고 → 로직을 설계하고 → 코드를 직접 짜는 방식입니다. 바이브 코딩은 반대입니다. 원하는 것을 자연어로 말하고, AI가 만든 결과물을 보고 느낌으로 피드백하며 완성해나갑니다.

개발자의 역할이 바뀝니다. 타이피스트가 아닌 감독이 됩니다.

"코드를 작성하는 것이 아니라, 원하는 것을 말하는 시대가 왔다.
바이브 코딩은 창의성과 기술 사이의 장벽을 허문다."

— Andrej Karpathy, 2025

02 / HOW IT WORKS
01
말로 설명하기

만들고 싶은 것을 일상 언어로 설명합니다. "따뜻한 느낌의 카페 예약 앱", "미니멀한 포트폴리오 사이트"처럼 느낌과 목적을 자유롭게 표현하세요.

NATURAL LANGUAGE
02
AI가 만들기

AI가 설명을 해석해 코드, 디자인, 구조를 생성합니다. 수십 초 안에 실제로 작동하는 결과물이 나옵니다. 코드를 이해할 필요가 없습니다.

AI GENERATION
03
느낌으로 수정

결과물을 보고 느낌으로 피드백합니다. "더 밝게", "버튼 크게", "딱딱한 느낌 줄여줘" — 감각적 언어가 그대로 변경사항이 됩니다.

VIBE FEEDBACK

바이브 코딩의
핵심 특징

기술적 배경 없이도 아이디어를 현실로 만들 수 있습니다. 중요한 것은 무엇을 만들고 싶은지 아는 것, 그뿐입니다.

💬
자연어 인터페이스

코드 대신 말로 명령합니다. 한국어, 영어 모두 가능. AI가 의도를 파악해 실행합니다.

⚡
즉각적인 결과

명령에서 결과까지 수십 초. 아이디어 → 프로토타입 사이클이 극적으로 단축됩니다.

🎨
감각적 피드백 루프

"이상해", "딱딱해", "더 가볍게" 같은 표현이 실제 수정으로 이어집니다. 느낌이 명령입니다.

🚀
원클릭 배포

만족스러운 결과물은 즉시 배포. 서버 설정, 도메인 연결 모두 자동으로 처리됩니다.

기존 방식 vs 바이브 코딩

03 / COMPARISON
BEFORE · 기존 개발
코드를 짠다
  • ✕언어 문법을 배워야 한다
  • ✕개발 환경 세팅에 시간 소요
  • ✕에러 디버깅에 수 시간 투자
  • ✕아이디어에서 결과까지 수 주
  • ✕디자인은 별도 전문가 필요
  • ✕배포 과정이 복잡하고 어렵다
AFTER · 바이브 코딩
감으로 만든다
  • ✓자연어로 설명하면 끝
  • ✓별도 설정 없이 즉시 시작
  • ✓"다시 해줘" 한 마디로 해결
  • ✓아이디어에서 결과까지 수 분
  • ✓느낌만 말하면 AI가 디자인
  • ✓원클릭으로 전 세계에 배포
